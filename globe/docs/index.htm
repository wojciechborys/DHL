Choosing earth.min.js or earth.core.min.js

earth.min.js bundles everything you need to start in one file.

earth.core.min.js is a stripped down version without:
- SVG Map: You need to provide a texture or use a custom SVG.
- Default Markers: You can use custom markers.
- CSS Styles: You need to load src/earth.css manually.

Gzip compression

Enable gzip compression on your web server to reduce the filesize of earth.min.js to about 260KB.

Asynchronous Loading

It is recommended to load the script asychronously. You can integrate a loading animation and fallback content.

Legacy Support (IE11)

Internet Explorer 11 is not supported by default and will show your fallback content. You can enable limited support for IE11.
IE11 doesn't support:
- SVG Maps: You need to use a texture as map or provide a fallback texture.
- Earth.mapHitTest

Get LatLng Geo Coordinates

You can use your favorite map service to get the lat/lng geo coordinates for a specific address.

Google Maps Example: You can get lat/lng coordinates directly from the address bar of your browser:
https://www.google.com/maps/place/New+York+City,+USA/@40.6918636,-74.0965785,11.25z/data=...
=>  { lat: 40.6918636, lng: -74.0965785 }

Using .obj 3d files as custom markers

Save your mesh as Wavefront (.obj) file in an uncompressed text format.
- Apply modifiers, triangulate faces
- Only meshes are supported, all other features are ignored (like materials, UVs, ...)
- The file can contain multiple objects/meshes at top level
- The object/mesh names should only consist of letters and numbers
- Your file should look like this:

o MyMesh
v 0.625 0.0 0.0
v 0.562 0.108 0.0
...
f 7 2 1
f 2 9 3
...
